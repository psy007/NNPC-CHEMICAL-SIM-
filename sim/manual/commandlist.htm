Command Line Interface Commands and Operators

 

Command

Meaning

cd <object path>

Changes the current object to the object described by the object path. This is a dot separated list of objects descending from the current object. The special items '/' represent the top object (the root flowsheet) and '..' represents the parent of the current object.

clear

 

destroys the current simulation and starts a new one

 

commonproperties <property list>

Sets or displays the list of properties that the property provider will calculate for each material port. If the list is present, then the current display property list will be replaced, otherwise the current list is displayed.

delete <object path>

Attempts to delete the object described by object path.

dir <object path>

Lists the objects contained in the object described by the object path. If the object path is omitted, the contents of the current object are listed.

displayproperties <property list>

If the property list is omitted, the properties that are displayed for a material port are listed. If the list is present, then the current display property list will be replaced.

go

 

resume simulation calculations, usually used after a hold command was issued

 

hold

stop simulation calculations. Useful to set up complex unit operation connections or change several simulation specifications without restarting calculations

language <language>

Switch to using 'language' for warning and information messages. Commands and variable names are not affected. The languages currently supported are English, Spanish and Portuguese.

log <file name>

Records all commands in a file. If the file already exists, the commands are appended to the end. If the file does not exist, it is created

ls <directory name>

Lists the files in the directory described by directory path, which is relative to the current directory. If no directory is given, the contents of the current directory are listed.

mkdir <directory name>

Creates the directory described by directory path, which is relative to the current directory.

quit

 

end a simulation session

 

read <file name>

 

read a simulation script. Note that you can use "/" (Unix style) or "\" (DOS Style) as your folder separator.

 

recall <file name>

 

read a simulation file. Same note as above

 

rm <file path>

removes file described by file path, which is relative to the current directory.

rmdir <directory path>

Removes the directory described by directory path, which is relative to the current directory.

store <file name>

 

store a simulation to disk. Same note as above

 

units <unit system>

Switches the default input and output units to 'unit system'. Unit systems can be added and customized, but the most common are SI, British and Field.

supportedproperties

Displays the list of properties that the property provider can calculate for each material port. The actual properties calculated are set with the commonproperties command.

valueOf <object.keyword>

Method to get the Value of a variable or an object. This is mainly used by interface writers. The command format is 'object.keyword'.
If keyword =
'key': returns the keys if object is a collection
'processvalue': returns the process value if object is a basicVariable or parameter
'path': returns the path of the object

=

 

Assignment operator used to set a variable value. Example:

 

Feed.In.T = 100 C

 

->

 

Connection operator used to attach ports. Example:

 

Feed.Out -> cooler.In

 

~=

 

Estimate operator. Used to set an estimated value. Example:

 

Feed.In.T ~= 100. This allows values to be automatically overwritten during convergence without creating consistency problems

 

.

 

Dereferencing operator. This allows you to transverse the object hierarchy. Example:

 

Feed.In.T allows you to access the Temperature BasicProperty object contained in the In material port object contained in the Feed Stream_Material object.

 

+

Depends on the left hand side object, but generally adds something to it, such as adding components to a thermo case.

-

Depends on the left hand side object, but generally subtracts something from it, such as removing components from a thermo case.

@

Alias operator - used to create an alias for the left hand side object using the right hand side description. Only current use is to alias contained unitop ports in parent object, thus creating a 'borrowed' port.

>>

Move operator - Used to move components within a thermo case or to fix the values in a port from another.

Examples:

  • thermo.hypo1* >> WATER # Move hypo1* in front of water
  • thermo.METHANE >> $ # Move methane to end
  • Feed.In >> V101.In # Use values in Feed.In to fix values in V101.In

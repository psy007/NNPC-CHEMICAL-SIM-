Multilanguage Support

 

You can add multilanguage support in Sim42 and replace the default English based messages from the simulator using any language you desire. In order to customize Sim42 for your language needs you may need to add a dictionary corresponding to your target language.

Initially locate the folder ..\sim\cmd\cmdlanguages. You will notice that Sim42 comes with support for three languages – English, Spanish and Portuguese. If you want to create a new dictionary please add your new Python file. You will need to add a method to your new file to support a dictionary:

def Messages():

    m = {}

    return m

And then you will have to create your dictionary (dictionary is a built-in data type in Python). Use the keys that exist in the English language as the base to create your own dictionary. English is the standard language of sim42 and has the most up to date message keys.

This is a small part of the Spanish dictionary for the Command Line Interface of sim42

 

def Messages():

    m = {}

    m['CMDBadOperator']         = "Operador desconocido %s"

    m['CMDCDInvalidObject']     = "No se puede cd al objeto %s"

    m['CMDInvalidUnitSet']      = "Set de unidades no válido %s"

    m['CMDInvalidContents']     = "No se puede obtener contenido de %s"

    m['CMDInvalidObject']       = "Objeto no válido %s"

    m['CMDUnknownObject']       = "Objeto desconocido %s"

    m['CMDNoCmpsMismatch']      = "Número incorrecto de composiciones"

    m['CMDAmbiguousUnit']       = "Unidad ambigua %s"

    m['CMDUnknownUnit']         = "Unidad desconocida %s"

   

    #... more messages go here

 

    return m

 

 

After you finish your dictionary you can add it to the list of supported Sim42’s languages by modifying the file __init__.py:

__all__ = ['English','Spanish',‘Portuguese’]

 

After creating a new dictionary for the command interpreter you will need to create a dictionary for the solver. Locate the folder ..sim\solver\languages and create a new Python file to support your desired language:

def Messages():

    """create dictionary of Spanish messages"""

    m = {}

    return m

and add the desired dictionary definition. The example below shows part of the Spanish solver dictionary.

def Messages():

    """create dictionary of Spanish messages"""

    m = {}

    m['BalanceInvalidPort']    = "Puerto no válido para balance(no materia o energía)"

    m['ConnectSameTypePorts']  = "Intento de conectar puertos de diferentes tipos en %s"

    m['CantAddToStage']        = "No se puede agregar %s a la etapa %d de %s"

    m['CantDeleteFromStage']   = "No se puede borrar %s de la etapa %d de %s"

    m['TowerQSpecError']       = "No se puede asignar flujo de energía a la etapa %d"

    m['CantAddToDraw']         = "No se puede agregar %s a la salida %s en la etapa %d de %s"

    m['InvalidTowerSpecPhase'] = "Fase no válida en la espec en la etapa %d de %s"

    m['InvalidDrawPhase']      = "Fase no válida para la salida en la etapa %d de %s"

    m['TooManyTowerSpecs']     = "%d especs encontradas, solo %d es necesaria en %s"

 

    #... more messages gohere

 

    return m

 

After you finish your dictionary you can add it to the list of supported Sim42’s languages by modifying the file __init__.py:

__all__ = ['English','Spanish', ‘Portuguese’]

 

Finally, after you have added your language you can instruct Sim42 to use it by issuing the following command:

 

/> language Spanish

 

 

The default language for Sim42 is English.
